## **M05 — Order Management**

Covers the full lifecycle of a placed order: confirmation, history, tracking, and admin fulfilment. Orders are created exclusively via the Stripe webhook (F-04.2) and progress through a defined status machine.

---

### **F-05.1 Order Confirmation & History**

**Summary**

Customers receive an order confirmation email immediately after payment, and can view their full order history and individual order details at any time from their account.

**Background / Context**

Order confirmation is the final trust moment of the purchase journey. Clear confirmation with order reference and delivery estimate reduces post-purchase anxiety and support contacts.

**User Story**

As a customer who just paid, I want to see a confirmation page and receive a confirmation email so that I know my order was received and when to expect delivery.

**Flow — Happy Path**

1. Stripe webhook (payment_intent.succeeded) received → order record created with status='confirmed'.
2. Customer redirected to /order-confirmation/:orderId with order summary.
3. TC-17: Confirmation email (E-04) dispatched within 4 hours (target: within seconds via BullMQ immediate job).
4. Customer visits /account/orders to view full order history — paginated, most recent first.
5. Customer clicks an order → /account/orders/:orderId shows itemised receipt, status, tracking link (when available).

**Flow — Edge Cases**

| Edge Case | Expected System Behaviour |
| :---- | :---- |
| Customer refreshes /order-confirmation after session cleared | Order found by orderId in URL. Displayed if customer is authenticated and owns the order. |
| Guest order (OD-01) — no account | Confirmation page accessible via one-time URL. Email sent. Order not accessible after session expires (no login to retrieve it). |
| Customer visits another customer's orderId | 403 — order not returned. |
| Webhook delayed — order not yet created when customer lands on confirmation | Page polls GET /orders/:orderId up to 30 seconds (3× at 10-second intervals). If still missing: 'Your order is being processed — check your email for confirmation.' |

**Business Rules**

| BR \# | Rule |
| :---- | :---- |
| BR-01 | Order history returns all orders for the authenticated customer, paginated (default 20 per page), sorted by created_at DESC. |
| BR-02 | Order detail includes: line items (product name, variant, qty, unit price, line total), subtotal, discount, shipping, tax, total, payment method last 4 digits, delivery address, status, tracking number (if dispatched). |
| BR-03 | TC-17: Confirmation email dispatched as an immediate BullMQ job on order creation. Retry: TC-29 (3× at 1 min, then 1× at 15 min). |
| BR-04 | Order reference number format: HS-[YEAR]-[6-digit sequential ID] (e.g. HS-2026-004821). |

**Input Validation Rules**

| Field | Type | Required | Validation Rules | Error Message Shown |
| :---- | :---- | :---- | :---- | :---- |
| orderId (path param) | UUID | Yes | Must be a valid UUID. Must belong to the authenticated customer. | 404 / 403 |
| page / limit (query params) | Integer | No | page ≥ 1. limit: 1–100, default 20. Invalid values default silently. | Silent default |

**UI / Display Rules**

| Condition / State | UI Component | Display Text | Colour / Style | Available Action |
| :---- | :---- | :---- | :---- | :---- |
| Order status = confirmed | Status badge | 'Order Confirmed' | Blue badge (\#185FA5) | 'Cancel Order' button (if within cancellation window) |
| Order status = processing | Status badge | 'Processing' | Amber badge (\#EF9F27) | — |
| Order status = dispatched | Status badge + tracking | 'Dispatched — Track your order' | Green badge (\#1D9E75) | Tracking link |
| Order status = delivered | Status badge | 'Delivered' | Green badge | 'Return Item' button (F-06.1) |
| Order status = cancelled | Status badge | 'Cancelled' | Grey badge | — |
| Cancellation window expired | 'Cancel Order' button state | Button hidden or shown as disabled with tooltip | Grey disabled | 'This order can no longer be cancelled' tooltip |
