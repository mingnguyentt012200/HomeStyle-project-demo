### **F-07.4 Admin Alerts**

**Summary**

Super Admins and Ops admins receive automated system alerts for critical events: payment failures, webhook errors, email dead-letter queue entries, and stock depletion.

**Permission Matrix**

| Method | Endpoint | Description | Guest | Customer | Ops | Content | Super Admin |
| :---- | :---- | :---- | :---- | :---- | :---- | :---- | :---- |
| **GET** | /api/v1/admin/alerts | List system alerts | ❌ | ❌ | ✅ | ❌ | ✅ |
| **PATCH** | /api/v1/admin/alerts/:alertId | Acknowledge alert | ❌ | ❌ | ✅ | ❌ | ✅ |
| **GET** | /api/v1/admin/alerts/settings | Get alert preferences | ❌ | ❌ | ❌ | ❌ | ✅ |
| **PATCH** | /api/v1/admin/alerts/settings | Update alert preferences | ❌ | ❌ | ❌ | ❌ | ✅ |

**API Contract**

| Method | Endpoint | Description | Request Body / Params | Response |
| :---- | :---- | :---- | :---- | :---- |
| **GET** | /api/v1/admin/alerts | List alerts | *?acknowledged=false&severity=critical\|warning&page=* | *200: { alerts[] }* |
| **PATCH** | /api/v1/admin/alerts/:alertId | Acknowledge | *{ acknowledged: true }* | *200: { alert }* |
| **PATCH** | /api/v1/admin/alerts/settings | Update preferences | *{ emailRecipients[], slackWebhook?, thresholds{} }* | *200: { settings }* |

**TC References:** TC-27 (low-stock threshold alert), TC-28 (back-in-stock), TC-29 (email failure alerts)

**Acceptance Criteria**

| \# | Criteria | Test Type | Status |
| :---- | :---- | :---- | :---- |
| AC-01 | Low-stock alert fired when stock_qty falls to or below admin-configured threshold. | Integration | — |
| AC-02 | Acknowledged alerts are hidden from default (unacknowledged) view. | Unit | — |
