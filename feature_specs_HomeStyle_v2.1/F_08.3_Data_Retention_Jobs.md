### **F-08.3 Data Retention Jobs**

**Summary**

Automated background jobs enforce data retention policies: purging unverified accounts, anonymising old orders, and archiving audit logs.

**Business Rules**

| BR \# | Rule |
| :---- | :---- |
| BR-01 | TC-31: Unverified accounts inactive for >30 days are soft-deleted by the nightly retention job. |
| BR-02 | TC-34: Orders older than 7 years are pseudonymised (customer PII replaced with hashed identifiers). Financial totals and line items retained. |
| BR-03 | TC-36: Audit logs older than 3 years are archived to cold storage (S3 Glacier). |
| BR-04 | All retention job runs are themselves logged with: job name, run timestamp, records affected, errors. |

**Permission Matrix**

| Method | Endpoint | Description | Guest | Customer | Ops | Content | Super Admin |
| :---- | :---- | :---- | :---- | :---- | :---- | :---- | :---- |
| **GET** | /api/v1/admin/retention/jobs | List retention job history | ❌ | ❌ | ❌ | ❌ | ✅ |
| **POST** | /api/v1/admin/retention/jobs/:jobName/run | Manually trigger retention job | ❌ | ❌ | ❌ | ❌ | ✅ |

**API Contract**

| Method | Endpoint | Description | Request Body / Params | Response |
| :---- | :---- | :---- | :---- | :---- |
| **GET** | /api/v1/admin/retention/jobs | Job history | *?jobName=&dateFrom=&page=* | *200: { runs[{ jobName, startedAt, completedAt, recordsAffected, errors[] }] }* |
| **POST** | /api/v1/admin/retention/jobs/:jobName/run | Manual trigger | *—* | *202: { jobId, status: 'queued' }* |

**TC References:** TC-31 (30-day unverified purge), TC-34 (7-year order pseudonymisation), TC-36 (3-year audit archive), TC-37 (retention job audit logging), TC-38 (GDPR supervisory authority reporting)

**Acceptance Criteria**

| \# | Criteria | Test Type | Status |
| :---- | :---- | :---- | :---- |
| AC-01 | Nightly job removes unverified accounts older than TC-31: 30 days. | Integration | — |
| AC-02 | Manual job trigger returns 202 immediately and processes asynchronously. | Integration | — |
| AC-03 | Each job run is logged with records affected count. | Integration | — |
