### **F-08.1 Cookie Consent**

**Summary**

GDPR-compliant cookie consent banner captures and records user consent before any non-essential cookies are set.

**Business Rules**

| BR \# | Rule |
| :---- | :---- |
| BR-01 | TC-31: No non-essential cookies set before consent is recorded. Consent recorded with timestamp and version. |
| BR-02 | Granular consent options: necessary (always on), analytics, marketing. |
| BR-03 | Consent preference stored in a first-party cookie (consent_preferences) and, if authenticated, synced to user record. |

**Permission Matrix**

| Method | Endpoint | Description | Guest | Customer | Ops | Content | Super Admin |
| :---- | :---- | :---- | :---- | :---- | :---- | :---- | :---- |
| **POST** | /api/v1/consent | Record consent choice | ✅ | ✅ | ✅ | ✅ | ✅ |
| **GET** | /api/v1/consent | Get current consent state | ✅ | ✅ | ✅ | ✅ | ✅ |
| **PATCH** | /api/v1/consent | Update consent preferences | ✅ | ✅ | ✅ | ✅ | ✅ |

**API Contract**

| Method | Endpoint | Description | Request Body / Params | Response |
| :---- | :---- | :---- | :---- | :---- |
| **POST** | /api/v1/consent | Record consent | *{ necessary: true, analytics: bool, marketing: bool, version: string }* | *200: { consentId, recordedAt }* |
| **GET** | /api/v1/consent | Get consent | *—* | *200: { necessary, analytics, marketing, recordedAt, version }* |
| **PATCH** | /api/v1/consent | Update preferences | *{ analytics?, marketing? }* | *200: { consent }* |

**TC References:** TC-31 (consent required before non-essential cookies), TC-32 (consent records retained 3 years for audit)
