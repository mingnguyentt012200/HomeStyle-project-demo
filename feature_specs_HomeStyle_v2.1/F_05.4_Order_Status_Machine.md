## **M05 — Order Management**

Covers the full lifecycle of a placed order: confirmation, history, tracking, and admin fulfilment. Orders are created exclusively via the Stripe webhook (F-04.2) and progress through a defined status machine.

---

### **F-05.4 Order Status Machine**

**Summary**

Defines and enforces all valid order status transitions. No direct DB writes bypass this machine.

**Background / Context**

Without explicit transition rules, bugs or admin errors can put orders into invalid states (e.g. dispatching a cancelled order). The state machine is the single source of truth for what status changes are permitted.

**Business Rules / Valid Transitions**

| From Status | Permitted Next Statuses | Triggered By |
| :---- | :---- | :---- |
| pending | confirmed, failed | Stripe webhook |
| confirmed | processing, cancelled | Admin (F-05.3) / Customer cancel (F-05.1) |
| processing | dispatched, cancelled | Admin (F-05.3) |
| dispatched | delivered | Admin or carrier webhook |
| delivered | return_requested | Customer (F-06.1) |
| failed | — (terminal) | Stripe webhook |
| cancelled | — (terminal) | Admin / Customer |
| return_requested | return_approved, return_rejected | Admin (F-06.2) |
| return_approved | refund_processing | System (F-06.3) |
| refund_processing | refunded | Stripe webhook |
| refunded | — (terminal) | — |
| return_rejected | — (terminal) | Admin (F-06.2) |
