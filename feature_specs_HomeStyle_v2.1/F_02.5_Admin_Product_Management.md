### **F-02.5 Admin Product Management**

**Summary**

Admin-Content users create, update, publish, archive, and manage inventory for all products and variants in the HomeStyle catalogue.

**User Story**

As an Admin-Content user, I want to create and publish products with all details (images, variants, pricing, stock) so that the catalogue is accurate and customers can purchase current offerings.

**Flow — Happy Path**

1. Admin navigates to /admin/products/new.
2. Admin fills in product details: name, description, category, materials, dimensions, weight.
3. Admin uploads images (up to 10, JPEG/PNG/WEBP, max 10MB each). S3 pre-signed upload URLs generated per image.
4. Admin adds variants (color/size/material combinations) with per-variant SKU, price, and stock_qty.
5. Admin clicks 'Publish' → product status set to 'active'. Search index updated within 60 seconds (BR-02 F-02.2).
6. Admin can later PATCH to update fields, adjust stock, or archive the product.

**Flow — Edge Cases**

| Edge Case | Expected System Behaviour |
| :---- | :---- |
| Admin publishes product with no variants | Allowed. Product treated as single-variant. |
| Stock adjustment takes quantity below 0 | 422: 'Stock quantity cannot be negative.' |
| Admin archives product with active orders | Product archived (no new purchases). Existing confirmed/processing orders unaffected. |
| Duplicate SKU | 422: 'This SKU already exists.' |

**Business Rules**

| BR \# | Rule |
| :---- | :---- |
| BR-01 | Products have statuses: draft, active, archived. Only active products visible to customers. |
| BR-02 | PATCH /admin/products/:productId is the partial-update endpoint. Only supplied fields are changed. |
| BR-03 | Stock adjustments via PATCH /admin/products/:productId/variants/:variantId are logged in stock_adjustment_log with admin ID, reason, before/after quantity. |
| BR-04 | Image uploads use S3 presigned PUT URLs. Images processed (resized/optimised) by a Lambda function on S3 event. |
| BR-05 | Archiving a product sets is_archived=true and removes it from the search index within 60 seconds. |

**Input Validation Rules**

| Field | Type | Required | Validation Rules | Error Message Shown |
| :---- | :---- | :---- | :---- | :---- |
| name | String | Yes | Min 3, max 200 chars. Plain text. | 'Product name is required' |
| description | String | Yes | Min 20, max 5000 chars. HTML allowed (sanitised: no script tags). | 'Description must be between 20 and 5000 characters' |
| categoryId | UUID | Yes | Must be an active leaf-level category. | 'Please select a valid category' |
| price | Decimal | Yes (per variant) | > 0. Max 2 decimal places. Max value: 999999.99. | 'Please enter a valid price greater than 0' |
| sku | String | Yes (per variant) | Alphanumeric + hyphens. Max 50 chars. Must be unique across all products. | 'SKU is required' / 'This SKU already exists' |
| stock_qty | Integer | Yes (per variant) | ≥ 0. Max 99999. | 'Stock quantity must be 0 or greater' |
| weight_kg | Decimal | No | > 0. Max 2 decimal places. Required for white-glove delivery eligibility calculation. | 'Weight must be greater than 0' |
| images | File[] | No | Max 10 per product. Each max 10MB. JPEG, PNG, WEBP only. | 'Each image must be under 10MB and in JPEG, PNG, or WEBP format' |

**API Contract**

| Method | Endpoint | Description | Request Body / Params | Response |
| :---- | :---- | :---- | :---- | :---- |
| **GET** | /api/v1/admin/products | List products | *?status=draft\|active\|archived&categoryId=&page=* | *200: { products[], total }* |
| **POST** | /api/v1/admin/products | Create product | *{ name, description, categoryId, weight_kg? }* | *201: { productId, status: 'draft' }* |
| **PATCH** | /api/v1/admin/products/:productId | Update product | *{ name?, description?, categoryId?, weight_kg?, metaTitle?, metaDescription? }* | *200: { product }* |
| **POST** | /api/v1/admin/products/:productId/publish | Publish | *—* | *200: { product, status: 'active' }* |
| **POST** | /api/v1/admin/products/:productId/archive | Archive | *—* | *200: { product, status: 'archived' }* |
| **POST** | /api/v1/admin/products/:productId/images/upload-url | Get upload URL | *{ filename, contentType }* | *200: { uploadUrl, imageId, expiresAt }* |
| **POST** | /api/v1/admin/products/:productId/variants | Add variant | *{ sku, price, stock_qty, attributes{} }* | *201: { variant }* |
| **PATCH** | /api/v1/admin/products/:productId/variants/:variantId | Update variant | *{ price?, stock_qty?, sku?, adjustmentReason? }* | *200: { variant }* |
