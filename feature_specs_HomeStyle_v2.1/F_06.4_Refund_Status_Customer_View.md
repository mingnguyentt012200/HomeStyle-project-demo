## **M06 — Returns & Refunds**

Handles the post-purchase return lifecycle: customer request, admin review, refund processing via Stripe, and status communication.

---

### **F-06.4 Refund Status (Customer View)**

**Summary**

Customers can check refund status from their order page without contacting support.

**Permission Matrix**

| Method | Endpoint | Description | Guest | Customer | Ops | Content | Super Admin |
| :---- | :---- | :---- | :---- | :---- | :---- | :---- | :---- |
| **GET** | /api/v1/orders/:orderId/refund | Get refund status | ❌ | ⚠ | ✅ | ❌ | ✅ |

**API Contract**

| Method | Endpoint | Description | Request Body / Params | Response |
| :---- | :---- | :---- | :---- | :---- |
| **GET** | /api/v1/orders/:orderId/refund | Refund status | *—* | *200: { refundId, amount, status, estimatedCreditDate }* |

**TC References:** TC-23 (5–10 business day credit window shown to customer)

**Acceptance Criteria**

| \# | Criteria | Test Type | Status |
| :---- | :---- | :---- | :---- |
| AC-01 | GET shows estimated credit date based on TC-23 window from refund initiation date. | Unit | — |
| AC-02 | Customer cannot view another customer's refund status — returns 403. | Security | — |
