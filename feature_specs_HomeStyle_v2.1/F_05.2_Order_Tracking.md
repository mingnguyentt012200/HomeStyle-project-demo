## **M05 — Order Management**

Covers the full lifecycle of a placed order: confirmation, history, tracking, and admin fulfilment. Orders are created exclusively via the Stripe webhook (F-04.2) and progress through a defined status machine.

---

### **F-05.2 Order Tracking**

**Summary**

Customers can view real-time carrier tracking status for dispatched orders, linked from their order detail page.

**Background / Context**

'Where is my order?' is the single most common customer service query for e-commerce. Self-service tracking eliminates these contacts.

**User Story**

As a customer waiting for my delivery, I want to see the latest tracking status directly from my order page so I don't need to contact support.

**Flow — Happy Path**

1. Admin marks order as dispatched and enters carrier + tracking number (F-05.3).
2. System updates order to status='dispatched'. TC-18: Dispatch email (E-05) sent.
3. Customer views order detail → 'Track your order' link shown with carrier name.
4. GET /orders/:orderId/tracking returns latest tracking events (fetched from carrier API or stored if cached).
5. Tracking events shown in reverse chronological order.

**Flow — Edge Cases**

| Edge Case | Expected System Behaviour |
| :---- | :---- |
| Carrier API unavailable | Last cached tracking data shown with timestamp. 'Tracking information may be delayed.' message shown. |
| Tracking number not yet active in carrier system | 'Tracking information is not yet available. Please check back in a few hours.' |
| Order not yet dispatched | Tracking endpoint returns 404. Tracking link not shown on order detail page. |

**Business Rules**

| BR \# | Rule |
| :---- | :---- |
| BR-01 | Tracking data cached in DB (order_tracking_events table) with TTL: refreshed every 2 hours via BullMQ job, or on-demand when customer views tracking. |
| BR-02 | Supported carriers configured in admin settings. Carrier API keys stored in environment variables (never in DB). |
| BR-03 | TC-18: Dispatch email sent within 48 hours of order confirmation (target: immediately on dispatch action). |
