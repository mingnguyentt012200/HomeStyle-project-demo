## **M06 — Returns & Refunds**

Handles the post-purchase return lifecycle: customer request, admin review, refund processing via Stripe, and status communication.

---

### **F-06.3 Refund Processing**

**Summary**

Approved returns trigger a Stripe refund (full or partial). Refund status is tracked via Stripe webhooks.

**Business Rules**

| BR \# | Rule |
| :---- | :---- |
| BR-01 | TC-22: Refund processed within 1 business day of admin approval. |
| BR-02 | TC-23: Refund appears on customer's card within 5–10 business days (Stripe SLA). |
| BR-03 | TC-21: Partial refunds permitted (e.g. returned subset of items). refundAmount set by admin during review. |
| BR-04 | Refund initiated via Stripe Refunds API using original PaymentIntent ID. |
| BR-05 | On refund.succeeded webhook: order status set to 'refunded'. Refund confirmation email sent. |

**Permission Matrix**

| Method | Endpoint | Description | Guest | Customer | Ops | Content | Super Admin |
| :---- | :---- | :---- | :---- | :---- | :---- | :---- | :---- |
| **POST** | /api/v1/admin/returns/:returnId/refund | Initiate Stripe refund | ❌ | ❌ | ✅ | ❌ | ✅ |
| **POST** | /api/v1/webhooks/stripe | Stripe refund webhook | ✅ | ✅ | ✅ | ✅ | ✅ |

**API Contract**

| Method | Endpoint | Description | Request Body / Params | Response |
| :---- | :---- | :---- | :---- | :---- |
| **POST** | /api/v1/admin/returns/:returnId/refund | Initiate refund | *{ refundAmount }* | *200: { refundId, status: 'refund_processing' }* |

**TC References:** TC-21 (partial refunds), TC-22 (1 business day processing), TC-23 (5–10 day card credit), TC-30 (refund audit retention)
