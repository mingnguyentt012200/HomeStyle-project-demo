## **M06 — Returns & Refunds**

Handles the post-purchase return lifecycle: customer request, admin review, refund processing via Stripe, and status communication.

---

### **F-06.1 Return Request**

**Summary**

Customers initiate a return for a delivered order item within the return window, selecting a reason and uploading optional photo evidence.

**User Story**

As a customer with a delivered order, I want to request a return online so that I can get a refund without calling support.

**Flow — Happy Path**

1. Customer visits /account/orders/:orderId and clicks 'Return Item' (visible only for status=delivered, within TC-19 return window).
2. Customer selects items to return, selects reason from dropdown, optionally uploads photo.
3. POST /orders/:orderId/returns creates return request with status='return_requested'.
4. Order status updated to 'return_requested' (F-05.4 transition).
5. Customer receives return confirmation email (E-08). Admin notified.

**Flow — Edge Cases**

| Edge Case | Expected System Behaviour |
| :---- | :---- |
| TC-19: Return window expired | 'Return Item' button hidden. POST returns 422: 'Return window has expired.' |
| Customer tries to return already-returned item | 422: 'A return request already exists for this item.' |
| Photo upload — file too large | 422: 'Image must be under 5MB.' |

**Business Rules**

| BR \# | Rule |
| :---- | :---- |
| BR-01 | TC-19: Return window is admin-configurable (default: 30 days from delivery). After window: endpoint returns 422. |
| BR-02 | Return reasons must match enum in DB (damaged, wrong_item, changed_mind, other). |
| BR-03 | Photo evidence is optional. Max 3 images, max 5MB each, JPEG/PNG/WEBP only. Stored in S3. |
| BR-04 | TC-20: Admin must review return request within 2 business days. |

**Input Validation Rules**

| Field | Type | Required | Validation Rules | Error Message Shown |
| :---- | :---- | :---- | :---- | :---- |
| lineItemIds | UUID[] | Yes | At least 1 item. All must belong to the orderId. All must be in a returnable state. | 'Please select at least one item to return' |
| reason | Enum | Yes | Must be one of: damaged, wrong_item, changed_mind, other. | 'Please select a return reason' |
| notes | String | No | Max 1000 chars. Plain text. | 'Notes must be 1000 characters or fewer' |
| photos | File[] | No | Max 3 files. Each max 5MB. JPEG, PNG, WEBP only. | 'Each image must be under 5MB and in JPEG, PNG, or WEBP format' |

**API Contract**

| Method | Endpoint | Description | Request Body / Params | Response |
| :---- | :---- | :---- | :---- | :---- |
| **POST** | /api/v1/orders/:orderId/returns | Submit return | *{ lineItemIds[], reason, notes?, photos[]? }* | *201: { returnId, status } / 422: window expired* |
| **GET** | /api/v1/orders/:orderId/returns | Return status | *—* | *200: { returnId, status, createdAt, reviewedAt?, refundAmount? }* |
