### **F-08.2 Data Erasure (Right to be Forgotten)**

**Summary**

Customers request deletion of their personal data. The system processes erasure within the TC-33 legally required window, with exemptions for financial records required by law.

**Business Rules**

| BR \# | Rule |
| :---- | :---- |
| BR-01 | TC-33: Erasure completed within 30 days of verified request. |
| BR-02 | Financial records (orders, invoices, tax records) retained for TC-34: 7 years per legal requirement — pseudonymised, not deleted. |
| BR-03 | Erasure covers: name, email, phone, addresses, wishlist, reviews (anonymised), session tokens. |
| BR-04 | Account closure (F-01.6) triggers erasure request automatically. |
| BR-05 | Erasure request must be confirmed by re-entering password. Email confirmation sent before processing begins. |

**Permission Matrix**

| **POST** | /api/v1/account/erasure-request | Submit erasure request | ❌ | ✅ | ❌ | ❌ | ❌ |
| **GET** | /api/v1/admin/erasure-requests | List erasure requests | ❌ | ❌ | ❌ | ❌ | ✅ |
| **PATCH** | /api/v1/admin/erasure-requests/:requestId | Process / complete erasure | ❌ | ❌ | ❌ | ❌ | ✅ |

**API Contract**

| Method | Endpoint | Description | Request Body / Params | Response |
| :---- | :---- | :---- | :---- | :---- |
| **POST** | /api/v1/account/erasure-request | Request erasure | *{ password }* | *200: { requestId, scheduledFor }* |
| **GET** | /api/v1/admin/erasure-requests | List requests | *?status=pending\|processing\|completed&page=* | *200: { requests[] }* |
| **PATCH** | /api/v1/admin/erasure-requests/:requestId | Process erasure | *{ status: 'processing'\|'completed', notes? }* | *200: { request }* |

**TC References:** TC-33 (30-day completion window), TC-34 (7-year financial record retention), TC-35 (erasure audit log retained 3 years)
