---

### **F-07.2 Review Moderation**

**Summary**

Admin-Content users review pending and flagged submissions, approve or reject them, and manage the product review feed.

**Permission Matrix**

| Method | Endpoint | Description | Guest | Customer | Ops | Content | Super Admin |
| :---- | :---- | :---- | :---- | :---- | :---- | :---- | :---- |
| **GET** | /api/v1/admin/reviews | List reviews (filterable) | ❌ | ❌ | ❌ | ✅ | ✅ |
| **PATCH** | /api/v1/admin/reviews/:reviewId | Approve / reject / flag | ❌ | ❌ | ❌ | ✅ | ✅ |
| **DELETE** | /api/v1/admin/reviews/:reviewId | Delete review | ❌ | ❌ | ❌ | ✅ | ✅ |

**API Contract**

| Method | Endpoint | Description | Request Body / Params | Response |
| :---- | :---- | :---- | :---- | :---- |
| **GET** | /api/v1/admin/reviews | List reviews | *?status=pending\|approved\|rejected\|flagged&productId=&page=* | *200: { reviews[], total }* |
| **PATCH** | /api/v1/admin/reviews/:reviewId | Moderate | *{ status: 'approved'\|'rejected', moderatorNotes? }* | *200: { review }* |
| **DELETE** | /api/v1/admin/reviews/:reviewId | Delete | *—* | *200: { message }* |

**TC References:** TC-25 (2 business day moderation SLA), TC-26 (review audit retention)

**Acceptance Criteria**

| \# | Criteria | Test Type | Status |
| :---- | :---- | :---- | :---- |
| AC-01 | PATCH approval makes review immediately visible on product page. | E2E | — |
| AC-02 | PATCH rejection sends rejection notification email to customer. | Integration | — |
| AC-03 | Ops admin role cannot access /admin/reviews — returns 403. | Security | — |
