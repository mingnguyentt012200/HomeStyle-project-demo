### **F-01.5 Token & Session Management**

**Summary**

Issues, rotates, and revokes JWT access tokens and refresh tokens for both customer and admin sessions. Provides the token refresh endpoint and handles session revocation on logout or security events.

**Background / Context**

Every authenticated feature depends on this module. Short-lived access tokens (15 min) limit blast radius if stolen. Rotating refresh tokens detect theft via reuse detection. This module is the security backbone — failures here break all authenticated flows.

**User Story**

As a registered customer or admin, I want my session to persist securely across browser sessions without repeatedly re-entering my password, and to be automatically logged out if my session is compromised.

**Flow — Happy Path**

1. On login (F-01.2 / F-01.4), server issues access token (httpOnly cookie) and refresh token (httpOnly cookie).
2. Frontend makes authenticated API calls using the access token.
3. When access token nears expiry, frontend calls POST /api/v1/auth/refresh.
4. Server validates refresh token. If valid: issues new access token and rotates refresh token (old invalidated).
5. On logout: POST /api/v1/auth/logout invalidates the refresh token family.

**Flow — Edge Cases**

| Edge Case | Expected System Behaviour |
| :---- | :---- |
| Refresh token already used (reuse detected) | Entire refresh token family revoked. User logged out on all devices. Security event logged. |
| Access token expired — no valid refresh token | 401 returned. Frontend redirects to /login. |
| Admin inactivity timeout (TC-05) | Server does not issue new tokens after TC-05: 30 min inactivity. Admin redirected to /admin/login. |
| User requests logout on all devices | DELETE /api/v1/auth/sessions revokes all refresh tokens for user. |

**Business Rules**

| BR \# | Rule |
| :---- | :---- |
| BR-01 | TC-01: Customer access token TTL: 15 minutes. TC-03: Admin access token TTL: 15 minutes. Stored in httpOnly, Secure, SameSite=Strict cookies. Never in localStorage or response body. |
| BR-02 | TC-02: Customer refresh token TTL: 30 days, rotating. TC-04: Admin refresh token TTL: 8 hours, rotating. |
| BR-03 | Refresh tokens are stored as bcrypt hashes in the DB (refresh\_tokens table), not plaintext. |
| BR-04 | Refresh token reuse (presenting an already-rotated token) triggers full token family revocation across all devices. Security event written to audit log. |
| BR-05 | On password reset (F-01.3): ALL refresh tokens for the user are immediately revoked regardless of device or session. |
| BR-06 | Token rotation is atomic: old token invalidated and new token issued in a single DB transaction. |

**Input Validation Rules**

| Field | Type | Required | Validation Rules | Error Message Shown |
| :---- | :---- | :---- | :---- | :---- |
| refreshToken (httpOnly cookie) | JWT string | Yes (for /auth/refresh) | Valid JWT signature. Not expired. Exists in refresh\_tokens table with is\_revoked=false. | 401: 'Session expired. Please log in again.' |
