## **M06 — Returns & Refunds**

Handles the post-purchase return lifecycle: customer request, admin review, refund processing via Stripe, and status communication.

---

### **F-06.2 Admin Return Review**

**Summary**

Ops admins review submitted return requests, approve or reject them, and trigger refund processing.

**Permission Matrix**

| Method | Endpoint | Description | Guest | Customer | Ops | Content | Super Admin |
| :---- | :---- | :---- | :---- | :---- | :---- | :---- | :---- |
| **GET** | /api/v1/admin/returns | List return requests | ❌ | ❌ | ✅ | ❌ | ✅ |
| **GET** | /api/v1/admin/returns/:returnId | Return detail | ❌ | ❌ | ✅ | ❌ | ✅ |
| **PATCH** | /api/v1/admin/returns/:returnId | Approve or reject | ❌ | ❌ | ✅ | ❌ | ✅ |

**API Contract**

| Method | Endpoint | Description | Request Body / Params | Response |
| :---- | :---- | :---- | :---- | :---- |
| **GET** | /api/v1/admin/returns | List returns | *?status=&dateFrom=&page=* | *200: { returns[], total }* |
| **GET** | /api/v1/admin/returns/:returnId | Return detail | *—* | *200: { return, photos[], order }* |
| **PATCH** | /api/v1/admin/returns/:returnId | Approve / reject | *{ decision: 'approved'\|'rejected', refundAmount?, adminNotes? }* | *200: { return } / 422: invalid decision* |

**TC References:** TC-20 (2 business day review SLA), TC-21 (partial refund rules)

**Acceptance Criteria**

| \# | Criteria | Test Type | Status |
| :---- | :---- | :---- | :---- |
| AC-01 | PATCH with decision='approved' triggers refund processing (F-06.3). | Integration | — |
| AC-02 | PATCH with decision='rejected' updates status and sends rejection email. | Integration | — |
| AC-03 | Customer role cannot access any /admin/returns endpoint. | Security | — |
