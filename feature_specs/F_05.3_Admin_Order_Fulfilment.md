## M05 --- Order Management

---

### F-05.3 Admin Order Fulfilment

**Summary**

Ops admins view the fulfilment queue, update order status, enter tracking numbers, coordinate white-glove delivery partners, and handle fulfilment exceptions.

**User Story**

As an Ops admin, I want to manage the order fulfilment queue so that orders are fulfilled accurately within SLA.

**Flow --- Happy Path**

1. Admin views /admin/orders.
2. 2. Admin selects confirmed order, changes status to 'processing'.
   3. 3. For made-to-order: admin records supplier order reference.
      4. 4. Admin enters carrier + tracking number, status updated to 'dispatched'. Dispatch email triggered.
         5. 5. For white-glove: admin notifies delivery partner and records appointment window.
           
            6. **Business Rules**
           
            7. - BR-01: Status transitions governed exclusively by F-05.4 Status Machine.
               - - BR-02: TC-17: 4-hour SLA to confirm order from placement. TC-18: 48-hour SLA to dispatch in-stock items.
                 - - BR-03: All admin order actions written to audit log.
                  
                   - **Permission Matrix**
                  
                   - | Method | Endpoint | Ops | Super Admin |
                   - |---|---|---|---|
                   - | GET | /api/v1/admin/orders | Yes | Yes |
                   - | PATCH | /api/v1/admin/orders/:orderId | Yes | Yes |
