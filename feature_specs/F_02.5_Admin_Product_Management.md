## M02 — Product Catalogue

---

### F-02.5 Admin Product & Catalogue Management

**Summary**

Admin-Content users manage the catalogue against HomeStyle's five-level hierarchy: Product Family → Product (parent) → Option Types (EAV attributes) → Variant / SKU → Bundle (Virtual SKU). Price, stock, weight, dimensions, and lead time live at the SKU level — never on the parent. Bundle availability is computed live from component ATP.

**Background / Context**

The catalogue model (BRD §17.1 / BR-PRD-01..10) is foundational. Flat product schemas break the moment we introduce flash sales, combos, or category expansion. This spec implements the senior-BA hierarchy with EAV attributes so the Phase 2 lighting / accessories range can be onboarded with zero schema change.

**User Story**

As an Admin — Content user, I want to create and publish products with their full configuration options, SKU-level prices, stock, dimensions, and specification PDFs so that customers see accurate ATP, pricing, and lead-time on every PDP and configurator combination.

---

**Catalogue Hierarchy (BRD §17.1 / BR-PRD-01..10)**

| Level | Entity | Owns | Example |
|---|---|---|---|
| 1 | Product Family / Collection | Navigation grouping, designer credit | "Aria by Marco Vitti", "Lounge Seating" |
| 2 | Product (Parent) | Shared metadata: description, materials list, base spec PDF | "Aria Lounge Chair" |
| 3 | Option Type (EAV) | Variant dimension definitions (per category) | Material, Finish, Fabric Grade, Base, Size |
| 4 | Variant / SKU | Price, stock, weight, **L×W×H**, lead time | `HS-CHAIR-001-WAL-BLK-FABA` |
| 5 | Bundle / Virtual SKU | Bundle marketing copy, bundle price | "Aria Lounge + Ottoman Set" |

---

**Flow — Happy Path (Single Product)**

1. Admin opens `/admin/catalogue/products/new`.
2. Admin assigns to one or more **Collections** (BR-PRD-07: multi-collection allowed).
3. Admin fills **parent product** fields: name, description, designer credit, materials overview, category (drives applicable EAV attribute schema per BR-PRD-08).
4. Admin uploads parent assets: up to 12 images (S3 presigned PUT, TC-09 pattern), one base specification PDF.
5. Admin opens **Configurator Builder**: selects which Option Types apply (e.g., Material + Finish + Fabric Grade + Base + Size). For each Option Type, defines valid values.
6. Admin generates SKU variants: system creates all valid combinations (BR-PRD-03: out-of-production combinations flagged at configurator level, not just hidden at cart).
7. For **each SKU variant**, admin enters: SKU code (auto-suggested per §10.1 assumption), price per currency (USD/EUR/GBP), stock_qty (initial), gross_weight_kg, **L × W × H** (mandatory non-zero per BR-PRD-04), lead_time_weeks, optional config-specific spec PDF.
8. Admin sets trade-pricing tier per SKU (BR-005, OD-05) — never visible to non-trade roles.
9. Admin moves status `Draft` → `Active`. Search index synced within 60 seconds (BC-02). Configurator combinations published.

**Flow — Bundle / Virtual SKU**

1. Admin creates parent Bundle record (e.g., "Aria Lounge + Ottoman Set").
2. Admin maps bundle to N component SKUs (e.g., 1× `HS-CHAIR-001-WAL-BLK-FABA` + 1× `HS-OTTO-001-WAL-BLK-FABA`).
3. Admin sets bundle price (typically discounted vs. component sum).
4. System computes `bundle_ATP = MIN(component_ATP)` at every reservation event (BR-PRD-05).
5. Selling a bundle decrements **each component's** Physical Stock individually (BR-PRD-06).

**Flow — Edge Cases**

| Edge Case | Expected System Behaviour |
|---|---|
| Admin saves SKU with L=0 or Weight=0 | 422 (BR-PRD-04): "Length, width, height, and gross weight are mandatory and must be greater than zero." Blocks publish. |
| Admin marks an option value (e.g., "Lacquered Steel") as out-of-production | All SKUs containing this value flagged at configurator level (BR-PRD-03). Customers cannot build this combination. Existing in-cart items retained until session expiry. |
| Bundle component goes out of stock | Bundle ATP auto-drops to 0 (BR-PRD-05). Bundle PDP shows "Notify Me" instead of Add to Cart. |
| Admin archives a Product (`is_archived=true`) | Product page 301-redirects to parent Collection immediately (BR-PRD-10). All variants removed from search index within 60 sec. Active wishlist holders notified once. |
| Admin attempts to delete (vs. archive) a Product with order history | Hard delete blocked: products with order references must be archived only (preserves order history integrity). |
| Two admins concurrently edit same SKU | Optimistic locking via `updated_at`. Second writer receives 409 with diff preview. |
| Admin uploads spec PDF >10MB | 413: "File exceeds 10MB. Please optimise the PDF." |

**Business Rules**

| BR # | Maps to BRD | Rule |
|---|---|---|
| F-02.5-BR-01 | BR-PRD-01 | Price, stock_qty, gross_weight_kg, L/W/H, and lead_time_weeks are stored at the **SKU (variant) level** only. Parent Product carries no pricing or inventory fields. |
| F-02.5-BR-02 | BR-PRD-02 | Every SKU has a unique system-generated SKU code that incorporates option-value codes (e.g., `HS-{ProductCode}-{MaterialCode}-{FinishCode}-{FabricCode}`). |
| F-02.5-BR-03 | BR-PRD-03 | Configurator must enforce **valid combinations only**. Out-of-production option values are flagged at the configurator level (visually struck-through with "Currently Unavailable" tooltip) — not just hidden at cart. |
| F-02.5-BR-04 | BR-PRD-04 | L, W, H, and gross weight are **mandatory non-zero** on every sellable SKU. Save is blocked otherwise — without dimensions, the carrier rate-quote API (BC-51) fails at checkout. |
| F-02.5-BR-05 | BR-PRD-05 | Bundle ATP = `MIN(component_ATP)` computed at every reservation event. A bundle is OOS the moment any single component is OOS. |
| F-02.5-BR-06 | BR-PRD-06 | Selling a bundle decrements **each component's** Physical Stock individually. Bundle is not a separate stock unit. |
| F-02.5-BR-07 | BR-PRD-07 | A product can belong to multiple Collections (e.g., "Lounge Seating" + "Aria by Marco Vitti"). |
| F-02.5-BR-08 | BR-PRD-08 | EAV attribute schema is extensible per category. New attributes (e.g., Phase 2 lighting: wattage, bulb type) can be added without core schema changes. |
| F-02.5-BR-09 | BR-PRD-09 | Specification PDFs default to parent product level; config-specific PDFs override at SKU level. Trade tier sees full-resolution version (BR-008). |
| F-02.5-BR-10 | BR-PRD-10 | Archive (not delete) is the only removal path. Sets `is_archived=true`; 301-redirects PDP to parent collection; clears search index within 60 sec. |
| F-02.5-BR-11 | BR-003 | Trade pricing per SKU is stored as a separate tier — never exposed in API response or page source to non-Trade roles. |
| F-02.5-BR-12 | — | All catalogue writes audit-logged with admin ID, before/after diff (extends BC-40). |

**Input Validation Rules**

| Field | Type | Required | Validation | Error Message |
|---|---|---|---|---|
| name (parent) | String | Yes | 3–200 chars | "Product name must be 3–200 characters" |
| description (parent) | String | Yes | 20–5000 chars; HTML sanitised | "Description must be 20–5000 characters" |
| designerId (parent) | UUID | No | Must reference valid designer | "Please select a valid designer" |
| collectionIds (parent) | UUID[] | Yes | ≥1 valid Collection | "Assign at least one Collection" |
| category (parent) | Enum | Yes | Determines applicable EAV attribute schema | "Please select a category" |
| sku (variant) | String | Yes | `[A-Z0-9-]+`, max 50, unique | "This SKU already exists" |
| price (variant, per currency) | Decimal | Yes | >0, max 2dp | "Enter a valid price" |
| stock_qty (variant) | Integer | Yes | ≥0, max 99999 | "Stock must be 0 or greater" |
| gross_weight_kg (variant) | Decimal | **Yes** | >0, max 2dp (BR-PRD-04) | "Gross weight is mandatory and must be > 0" |
| length_cm / width_cm / height_cm (variant) | Decimal | **Yes** | >0, max 1dp (BR-PRD-04) | "Dimensions are mandatory and must be > 0" |
| lead_time_weeks (variant) | String | No | `X` or `X–Y weeks`, max 50 chars | "Enter a valid lead time" |
| safety_stock (variant) | Integer | No | ≥0, defaults per BR-INV-02 | "Safety stock cannot be negative" |
| low_stock_threshold (variant) | Integer | No | ≥0, default 5 (BR-INV-10) | — |
