### F-01.2 Customer Login (Email + Social OAuth)

**Summary**

Registered customers authenticate via email/password or Google/Facebook OAuth 2.0, receiving a short-lived JWT access token and rotating refresh token.

**Background / Context**

Login is the gateway to all authenticated features — order history, wishlist, trade pricing, and checkout. Social login removes the password barrier for a large design-savvy user segment accustomed to convenience. The JWT strategy (short-lived access + rotating refresh) protects against token theft while minimising re-login friction.

**User Story**

As a registered customer, I want to log in with my email/password or Google account so that I can access my orders, project wishlist, trade pricing, and checkout without re-entering my details every time.

**Flow — Happy Path**

1. User navigates to /login, enters email and password, clicks 'Sign In'.
2. 2. Server validates credentials (bcrypt.compare). Checks status='verified' and is_active=true.
   3. 3. Server issues TC-01: 15-minute JWT access token (httpOnly cookie) and TC-02: 30-day rotating refresh token (httpOnly cookie, Secure, SameSite=Strict).
      4. 4. User redirected to account page or returnUrl param destination.
        
         5. **Flow — Edge Cases**
        
         6. | Edge Case | Expected System Behaviour |
         7. |---|---|
         8. | Wrong password (< 5 attempts) | Generic error: 'Incorrect email or password.' No indication of which field is wrong (prevents enumeration). |
         9. | 5 consecutive failed attempts (TC-10) | Account locked 15 minutes. Show countdown: 'Too many failed attempts. Try again in X minutes.' |
         10. | Account not verified | 'Please verify your email before logging in.' Link to resend verification. |
         11. | Account deactivated by admin | 'This account has been deactivated. Contact support if you believe this is an error.' |
         12. | OAuth state token expired (TC-08) | Generic 'Sign in failed' error. User must restart OAuth flow. |
         13. | OAuth callback — state mismatch | Reject with 400. Log security event. |
         14. | Social login with existing email/password account | Link the OAuth provider ID to the existing account. No duplicate account created. |
        
         15. **Business Rules**
        
         16. | BR # | Rule |
         17. |---|---|
         18. | BR-01 | TC-01: Access token lifetime: 15 minutes. Stored in httpOnly cookie — never exposed in localStorage or response body for client-side JS access. |
         19. | BR-02 | TC-02: Refresh token lifetime: 30 days. Rotation on every use — old token immediately invalidated. Reuse of a revoked token triggers full session family revocation. |
         20. | BR-03 | TC-10: Lockout after 5 consecutive failed password attempts. Duration: 15 minutes. Counter stored in Redis with TTL. Resets on successful login. |
         21. | BR-04 | Error messages for failed login must never indicate whether email OR password was wrong. |
         22. | BR-05 | TC-08: OAuth state token: random UUID per login attempt, stored in Redis with 10-minute TTL. Mismatch on callback = reject (CSRF protection). |
         23. | BR-06 | Google/Facebook login disabled on EU-facing routes until Legal confirms GDPR legal basis (OD-06 dependency). |
        
         24. **Input Validation Rules**
        
         25. | Field | Type | Required | Validation Rules | Error Message Shown |
         26. |---|---|---|---|---|
         27. | email | String (email) | Yes | Valid RFC 5322 format. Max 254 characters. Trimmed and lowercased before lookup. | 'Please enter a valid email address' |
         28. | password | String | Yes | Presence check only — strength not re-validated at login. Max 128 characters. Sent over HTTPS only. | 'Password is required' |

         46. **API Contract**
        
         47. | Method | Endpoint | Description | Request Body / Params | Response |
         48. |---|---|---|---|---|
         49. | POST | /api/v1/auth/login | Email/password login | { email, password } | 200: { user } + httpOnly cookies / 401 / 423: locked |
         50. | GET | /api/v1/auth/google | Initiate Google OAuth | — | 302: redirect to Google |
         51. | GET | /api/v1/auth/google/callback | Google OAuth callback | ?code=&state= | 302: redirect with cookies / 400: CSRF fail |
         52. | POST | /api/v1/auth/logout | Invalidate refresh token | — (httpOnly cookie) | 200: { message } |
