### F-01.4 Admin Login & Two-Factor Authentication

**Summary**

Secure admin panel access with mandatory TOTP-based 2FA, strict rate limiting, and inactivity auto-logout.

**Background / Context**

Admin users have elevated privileges — product catalogue, order processing, customer data, trade account approvals, and financial reporting. A compromised admin account is catastrophic for both business operations and customer trust in a premium brand context. 2FA is non-negotiable. Stricter limits than customer login apply throughout.

**User Story**

As an admin user, I want to log in securely with email, password, and 2FA code so that the admin panel is protected even if my password is compromised.

**Flow — Happy Path**

1. Admin navigates to /admin/login, enters email and password, clicks 'Continue'.
2. 2. Server validates credentials. If valid: returns intermediate state (requiresTOTP: true). No tokens issued yet.
   3. 3. Admin opens authenticator app, enters 6-digit TOTP code.
      4. 4. Server validates TOTP: current 30-second window ± 1 window (clock skew). Code cannot be reused within same window.
         5. 5. Server issues TC-03: 15-min admin JWT and TC-04: 8-hour admin refresh token.
            6. 6. Admin redirected to /admin/dashboard.
               7. 7. TC-05: Inactivity timer starts. At 5 minutes remaining: modal warning shown.
                 
                  8. **Flow — Edge Cases**
                 
                  9. | Edge Case | Expected System Behaviour |
                  10. |---|---|
                  11. | 3 consecutive failed attempts (TC-11) | Account locked 30 minutes. Super Admin notified by email. |
                  12. | Correct password but wrong TOTP | 401 — tokens NOT issued at password step. Counts toward TC-11 lockout. |
                  13. | TC-05: Inactivity reaches 30 minutes | Session terminated. Admin redirected to login. Admin panel auto-saves drafts every 60s. |
                  14. | Admin tries to reuse a TOTP code | Rejected — each 30-second window code is single-use. |
                 
                  15. **Business Rules**
                 
                  16. | BR # | Rule |
                  17. |---|---|
                  18. | BR-01 | 2FA is mandatory for ALL admin accounts. New admins cannot log in until 2FA is enrolled via /admin/auth/setup-2fa. |
                  19. | BR-02 | TC-11: Lockout after 3 consecutive failed attempts (password or TOTP). Duration: 30 minutes. Super Admin receives email on lockout. |
                  20. | BR-03 | TC-03: Admin access token: 15 minutes. TC-04: Admin refresh token: 8 hours (shorter than customer due to elevated privileges). |
                  21. | BR-04 | TC-05: Admin inactivity timeout: 30 minutes, detected via frontend heartbeat. 5-minute warning modal before termination. |
                  22. | BR-05 | Admin panel must auto-save all open form drafts to localStorage every 60 seconds to prevent data loss on session expiry. |
                  23. | BR-06 | Super Admin can manually unlock a locked admin account via the admin user management panel. |
                 
                  24. **Input Validation Rules**
                 
                  25. | Field | Type | Required | Validation Rules | Error Message Shown |
                  26. |---|---|---|---|---|
                  27. | email | String (email) | Yes | Valid email format. Looked up against admin_users table only. | 'Please enter a valid email address' |
                  28. | password | String | Yes | Presence check only at login. | 'Password is required' |
                  29. | totpCode | String (numeric) | Yes (step 2) | Exactly 6 digits. Numeric only. Current window ± 1 window. Single-use per window. | 'Enter the 6-digit code from your authenticator app' / 'Invalid or expired code.' |
