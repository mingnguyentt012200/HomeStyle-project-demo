## M09 — Admin Operations (Logistics & Commerce Configuration)

---

### F-09.4 Admin Dashboard & KPI Cards

**Summary**

Role-aware admin landing page that aggregates real-time KPI cards (today's revenue, order count, new customers, low-stock alerts, pending trade applications) and short-period sales-trend charts. Each card is permission-scoped: an Admin — Content user does not see revenue; an Admin — Trade does not see fulfilment SLA. Acts as the daily situational-awareness surface and the launch point into deeper modules.

**Background / Context**

BRD §8.4 explicitly requires an admin dashboard with real-time KPI cards and a sales-trend chart. BRD §19 enumerates the KPI catalogue (conversion, AOV, trade-account activation, return rate, etc.). This spec wires the two together and codifies the role-based card visibility so a single dashboard implementation serves all admin roles.

**User Story**

As an admin (any role), I want a single landing page that shows me what needs attention today — revenue, fulfilment backlogs, trade applications, alerts — so that I can prioritise my work without trawling individual modules.

**Flow — Happy Path**

1. Admin logs in (F-01.4) → redirected to `/admin/dashboard` (SCR-A-12 / role-scoped variants).
2. UI hits `/api/v1/admin/dashboard?role=<role>` — server returns the cards + chart series the caller is allowed to see.
3. KPI cards render: each card shows headline value, delta vs prior period, sparkline.
4. Below cards: **Sales Trend** chart (default 30 days; switchable to 7/30/90 days). Series: revenue, orders, AOV.
5. **Action shelf**: queue counts that deep-link into modules — Pending trade applications → F-10.1, Pending returns → F-06.2, Low-stock SKUs → F-09.3, Pending review moderation → F-07.2, Overdue orders (TC-17/TC-18 breach) → F-05.3.
6. Cards refresh every 60s via SSE or polling. Manual refresh button available.

**Flow — Edge Cases**

| Edge Case | Expected System Behaviour |
|---|---|
| Role lacks visibility for a card | Card omitted server-side; UI never renders empty placeholder. |
| KPI source is degraded (e.g. analytics warehouse lag) | Card shows last-known value with `as of HH:MM` timestamp and an amber dot. |
| Sales-trend chart range exceeds retention | Server clamps to max retention window; UI shows informational footnote. |
| Currency mix (multi-currency orders) | Headline revenue shown in store base currency (USD by default). Footnote: "Converted at order-time FX rate". |
| Newly-created admin with no past activity | Welcome state replaces personalised "needs attention" copy; cards still render. |

**Business Rules**

| BR # | Rule |
|---|---|
| BR-01 | Card visibility is enforced server-side (allow-list per role), not by hiding in the UI. |
| BR-02 | Revenue is reported net of refunds and trade discount, gross of tax/shipping (consistent with BRD §19 KPI definitions). |
| BR-03 | "Today" boundaries are based on the store's configured business timezone (Super Admin setting), not the admin's browser timezone. |
| BR-04 | Sparkline trend windows: 7-day, 30-day, 90-day. Each card declares its native window. |
| BR-05 | Trade application count includes only `submitted` and `under_review` states (not `approved` or `rejected`). |
| BR-06 | Low-stock count uses F-09.3 BR-06 threshold logic. Discontinued SKUs excluded. |
| BR-07 | Cache TTL 60s. Manual refresh bypasses cache and forces recompute. |
| BR-08 | All dashboard reads are audited at request granularity (caller, timestamp) for security review; payloads are not logged. |

**Permission Matrix**

| Method | Endpoint | Description | Guest | Customer | Trade | Ops | Content | Trade Admin | Super Admin |
|---|---|---|---|---|---|---|---|---|---|
| **GET** | /api/v1/admin/dashboard | Role-scoped dashboard payload | ❌ | ❌ | ❌ | ✅ | ✅ | ✅ | ✅ |
| **GET** | /api/v1/admin/dashboard/sales-trend | Sales-trend series | ❌ | ❌ | ❌ | ✅ | ❌ | 👁 (B2B only) | ✅ |

**API Contract**

| Method | Endpoint | Description | Request Body / Params | Response |
|---|---|---|---|---|
| **GET** | /api/v1/admin/dashboard | Get cards | `?refresh=` | `200: { cards[], asOf, role }` |
| **GET** | /api/v1/admin/dashboard/sales-trend | Time-series | `?range=7d\|30d\|90d&segment=all\|b2c\|b2b` | `200: { series[{date, revenue, orders, aov}] }` |
