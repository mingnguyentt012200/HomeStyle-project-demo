## M09 — Admin Operations (Logistics & Commerce Configuration)

---

### F-09.1 Admin Shipping Configuration

**Summary**

Ops admins (and Super Admin) configure shipping zones, flat-rate and weight-based shipping rates, free-shipping thresholds, and white-glove delivery partners. The runtime checkout calculator (F-04.4) reads the resulting `shipping_rates`, `shipping_zones`, and `whiteglove_partners` tables to compute customer-facing shipping cost and delivery options.

**Background / Context**

HomeStyle ships a heterogeneous mix of small accessories (parcel carriers), medium furniture (threshold/2-person delivery) and large furniture (LTL + white-glove) across US and EU markets (BRD §18.2). Rates change as carrier contracts are renegotiated, fuel surcharges shift, and new white-glove partners are onboarded for new ZIP/postal regions. Without an admin UI, every change is a code/DB ticket — unacceptable at premium-furniture scale. F-04.4 is the *consumer* of this data; F-09.1 is the *producer*.

**User Story**

As an Ops admin, I want to define shipping zones, weight-based rates, and white-glove partners so that checkout shows accurate shipping costs and eligibility without engineering involvement.

**Flow — Happy Path**

1. Admin opens `/admin/shipping` (SCR-A-11). Three tabs: **Zones**, **Rates**, **White-Glove Partners**.
2. **Zones tab**: admin creates a zone (e.g. "EU-DE-North"), assigns ISO country code + postal/ZIP prefixes, sets currency.
3. **Rates tab**: admin selects a zone and adds rate row(s): `delivery_method` (parcel | threshold | white-glove), `flat_rate`, `per_kg_rate`, `min_weight_kg`, `max_weight_kg`, `free_shipping_threshold` (optional), `eta_days_min`, `eta_days_max`.
4. **White-Glove Partners tab**: admin creates partner record — name, contact, service ZIP/postal prefixes, surcharge amount, capabilities (room-of-choice, packaging removal, assembly).
5. Admin clicks **Save**. System validates rate ranges do not overlap within the same zone+method, and white-glove ZIP coverage does not duplicate.
6. On save, configuration is versioned (audit log, F-13.x). New rates take effect for *new* checkout sessions only — in-flight sessions retain their locked rate per TC-14.

**Flow — Edge Cases**

| Edge Case | Expected System Behaviour |
|---|---|
| Admin creates overlapping weight ranges in the same zone+method | 422 returned. UI highlights overlapping rows. |
| Admin deletes a zone with active orders referencing it | Soft-delete only. Zone marked `archived` and excluded from new lookups; historical orders retain reference. |
| Admin deletes a white-glove partner with scheduled deliveries | Block delete. Show count of scheduled deliveries; require reassignment first. |
| Free-shipping threshold set to 0 or negative | 422: 'Threshold must be greater than zero, or leave blank to disable.' |
| Admin uploads partner ZIP coverage CSV with invalid postcodes | Per-row validation report; valid rows imported, invalid rows rejected with reasons. |
| Admin changes rate that affects in-flight checkout sessions | Locked sessions unaffected (TC-14, BR-04). Banner warns admin: "Currently 23 active checkout sessions on prior rate." |

**Business Rules**

| BR # | Rule |
|---|---|
| BR-01 | Shipping rate formula: `flat_rate + (item_weight_kg × per_kg_rate)`, summed across line items, with white-glove surcharge added once if any line item triggers white-glove (per F-04.4 BR-05). |
| BR-02 | Zones are uniquely identified by `(country_code, postal_prefix_pattern)`. A postal prefix may belong to only one zone per delivery method. |
| BR-03 | White-glove eligibility is a function of (a) at least one line item with `requires_whiteglove=true` (large furniture per BRD §18.2) AND (b) destination ZIP/postal matches an active white-glove partner coverage. If (a) without (b): UI offers "threshold delivery" or "Contact us" per BRD §18.2. |
| BR-04 | Rate changes are non-retroactive. Locked checkout sessions (TC-14) and confirmed orders use the rate captured at lock time. |
| BR-05 | Only Admin — Operations and Super Admin can write to shipping configuration. Admin — Content has no access. Admin — Trade has read-only access (needed to advise B2B customers). |
| BR-06 | Free-shipping threshold is per zone+method. When met, F-04.4 shows `$0.00 — Free Shipping (orders over $X)`. |
| BR-07 | All write operations are written to the audit log with admin ID, timestamp, before/after diff. |
| BR-08 | Soft-delete only for zones, rates, and partners. Hard-delete forbidden to preserve historical order traceability. |

**Input Validation Rules**

| Field | Type | Required | Validation Rules | Error Message Shown |
|---|---|---|---|---|
| zone_code | String | Yes | Unique per environment. `[A-Z]{2}-[A-Z0-9-]+`, max 32 chars. | 'Zone code must be uppercase, format: `XX-Region`.' |
| country_code | ISO-3166-1 alpha-2 | Yes | Must be in supported countries list. | 'Country not supported. Contact Super Admin.' |
| postal_prefix | String | Yes | 1–10 chars; regex per country (US: 5-digit numeric prefix; UK: outward code; etc.). | 'Invalid postal/ZIP format for selected country.' |
| flat_rate | Decimal(10,2) | Yes | ≥ 0; currency matches zone currency. | 'Flat rate cannot be negative.' |
| per_kg_rate | Decimal(10,2) | Yes | ≥ 0. | 'Per-kg rate cannot be negative.' |
| min_weight_kg / max_weight_kg | Decimal(10,2) | Yes | min ≥ 0; max > min; no overlap with other rows in same zone+method. | 'Weight range overlaps with existing rate row.' |
| free_shipping_threshold | Decimal(10,2) | No | If set, > 0. | 'Threshold must be greater than zero.' |
| eta_days_min / eta_days_max | Integer | Yes | min ≥ 1; max ≥ min; max ≤ 60. | 'ETA must be between 1 and 60 days.' |
| whiteglove_surcharge | Decimal(10,2) | Yes | ≥ 0. | 'Surcharge cannot be negative.' |
