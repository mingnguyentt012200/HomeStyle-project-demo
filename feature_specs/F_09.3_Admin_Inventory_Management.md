## M09 — Admin Operations (Logistics & Commerce Configuration)

---

### F-09.3 Admin Inventory & Stock Management

**Summary**

Admin — Operations users manage the three-layer inventory model — **Physical Stock**, **Reserved Stock**, and **Available to Promise (ATP)** — with explicit Soft Lock / Hard Lock semantics. The runtime catalogue (F-02.3), checkout (F-04.x), routing (F-05.5), and pick/pack (F-05.6) all read these layers to drive PDP availability, reservations, and dispatch SLAs (TC-18).

**Background / Context**

Inventory is a multi-layered state machine, not a single counter (BRD §17.4). The single biggest cause of overselling at peak traffic is treating "stock" as one number. This spec implements the senior-BA approach: ATP is the only number shown to customers (BR-INV-01), Soft Locks (TC-13 Redis SETNX) protect the checkout window, Hard Locks (durable DB rows) protect the paid-but-unshipped window, and Safety Stock absorbs carrier and channel lag (BR-INV-02).

**User Story**

As an Admin — Operations user, I want to view and adjust the three inventory layers (Physical / Reserved / ATP) with audited reasons, manage safety stock and low-stock thresholds, and investigate oversell incidents — so that customers never see stale ATP and dispatch SLAs are reliably met.

---

**Inventory Model (BRD §17.4 / BR-INV-01..10)**

| Layer | Definition | Mutator |
|---|---|---|
| **Physical Stock** | Goods physically on shelves. Ground truth. | F-05.6 pick confirmation (decrement); F-09.3 GRN / cycle count / manual adjustment (increment / decrement). |
| **Reserved Stock** | Active Soft Locks + Hard Locks against this SKU. | F-04.1 checkout init (Soft Lock); F-04.2 payment confirmation (promote to Hard Lock); F-05.6 pick (release Hard Lock); F-05.3 cancellation (release). |
| **ATP — Available to Promise** | `ATP = Physical − Reserved − SafetyStock`. The only number on PDPs. | Derived; recomputed on every Reserved-changing event. |

**Locks**

| Lock | Storage | TTL | Released by |
|---|---|---|---|
| **Soft Lock** | Redis SETNX | TC-13 (15 min) | TTL expiry / cart abandon / payment fail |
| **Hard Lock** | DB row `order_allocations` | None | Pick confirmation / order cancel / Super Admin manual override |

---

**Flow — Happy Path (Stock Adjustment)**

1. Admin opens `/admin/inventory` (SCR-A-10). Default view lists SKUs filtered by low-stock, recent movement, OOS, made-to-order, or oversell-flagged.
2. Admin clicks a SKU → **Inventory Detail**: Physical, Reserved (split: Soft Lock count / Hard Lock count), ATP, Safety Stock, Low-Stock Threshold, recent movements (audit log).
3. Admin chooses **Adjust Stock** → enters delta (signed integer) + mandatory reason (controlled vocab) + optional note.
4. System validates: cannot drive Physical < (current Hard Locked qty) — that would be an immediate oversell.
5. Adjustment commits atomically. `ATP` recomputed. If `ATP` transitions OOS → InStock and a "Notify Me" list exists for this SKU: TC-28 hourly job queued for back-in-stock notification.
6. Adjustment logged in audit (BC-40, BR-INV-08).
7. If ATP crosses Low-Stock Threshold downward: Process 11.7 low-stock alert generated (subject to TC-27: 24-hr cooldown).

**Flow — Happy Path (Safety Stock Review)**

1. Admin (quarterly per **TC-INV-03**) opens **Safety Stock Review** report.
2. Report ranks SKUs by oversell incidents, near-oversell events, carrier-delay return events, and inbound-sync mismatches.
3. Admin updates Safety Stock per SKU (defaults per BR-INV-02: 2 for parcel accessories; 1 for white-glove; 0 for made-to-order).
4. Changes audited; effective immediately at next ATP recomputation.

**Flow — Happy Path (Oversell Incident Triage)**

1. System detects oversell at F-05.6 pick step (Physical < total Hard Locked). Per BR-INV-09: last-paid order auto-moved to `On-Hold`; E-OVERSELL email dispatched.
2. Admin opens `/admin/inventory/oversell-incidents` — list of open incidents.
3. Admin investigates root cause: race condition (system flaw), safety stock too low (config), supplier shortage (operational), cycle-count variance (warehouse).
4. Admin marks incident `Resolved` with root-cause tag. Used to drive monthly Oversell Rate KPI (target ≤0.05%; postmortem if >0.2%).

**Flow — Edge Cases**

| Edge Case | Expected System Behaviour |
|---|---|
| Admin attempts to set Physical < Hard Locked qty | 422 (BR-INV-05): "Cannot reduce Physical below committed Hard Locks (N units). Cancel orders first or contact Super Admin." |
| Cycle count enters negative count | Allowed with `count_correction` reason. System flags SKU for investigation; new ATP recomputed. |
| Admin sets `discontinued` on a SKU with active Hard Locks | Blocked. Requires (a) cancelling those orders or (b) Super Admin override. Soft Locks left to expire naturally. |
| Concurrent admin edit | Optimistic locking via `updated_at`. Second writer 409 with diff preview. |
| Bulk CSV: 5,000 rows, mixed valid / invalid | Per-row validation report; valid rows committed atomically; invalid rows skipped with error column. |
| Customer order cancel happens during admin adjustment | DB transaction serialises; final ATP correct in either order. |
| Channel buffer change (Phase 3 marketplace) | Reduces ATP feed to that channel only; internal ATP unaffected. |

**Business Rules**

| BR # | Maps to BRD | Rule |
|---|---|---|
| F-09.3-BR-01 | BR-INV-01 | `ATP = Physical − Reserved − SafetyStock`. ATP is the **only** number shown to customers and used for Add-to-Cart eligibility. |
| F-09.3-BR-02 | BR-INV-02 | Safety Stock per SKU. Defaults: 2 (parcel accessory), 1 (white-glove), 0 (made-to-order). Configurable via System Settings (BC-41). |
| F-09.3-BR-03 | BR-INV-03 | Soft Lock created on checkout initiation (TC-13: Redis SETNX, 15-min TTL). Released on TTL expiry, cart abandon, or payment failure. |
| F-09.3-BR-04 | BR-INV-04 | Hard Lock created on `payment_intent.succeeded` (DB row in `order_allocations`). Released **only** on pick confirmation, order cancel, or admin manual override (audit-logged). |
| F-09.3-BR-05 | BR-INV-05 | Reservation must be atomic. Redis SETNX (Soft) and DB transaction (Hard) prevent concurrent reservation of the same physical unit. Cannot set Physical below committed Hard Locks. |
| F-09.3-BR-06 | BR-INV-06 | Made-to-Order SKUs have Physical = 0 but remain orderable. Their "reservation" is a supplier-production commitment, not a physical-stock lock. |
| F-09.3-BR-07 | BR-INV-07 | Returned items enter `QC-Hold` virtual location and do **not** increment Physical until QC pass. QC fail routes to scrap or Phase 3 refurb channel. |
| F-09.3-BR-08 | BR-INV-08 | Every adjustment outside the order flow (cycle count, supplier delivery, manual write-off) is audit-logged with actor, before/after, reason code. |
| F-09.3-BR-09 | BR-INV-09 | Oversell Rate KPI ≤0.05%. >0.2% in any rolling month triggers mandatory postmortem and Safety Stock review. |
| F-09.3-BR-10 | BR-INV-10 | Low-stock threshold per SKU configurable. Defaults: 5 (parcel), 2 (white-glove). PDP shows "Only X left" below threshold. Alert subject to TC-27: 24-hr cooldown per SKU. |
| F-09.3-BR-11 | BR-WHS-10 | Buffer Stock (channel sync, Phase 3) is separate from Safety Stock. Withholds N units from each external channel's ATP feed; internal ATP unaffected. |
| F-09.3-BR-12 | — | Admin — Content has read-only access to SKU state and lead-time copy via F-02.5; cannot mutate inventory. |

**Input Validation Rules**

| Field | Type | Required | Validation | Error Message |
|---|---|---|---|---|
| sku | String | Yes | Must exist; `[A-Z0-9-]+`; max 50 | "SKU not recognised" |
| adjustment_qty | Integer | Yes | Non-zero; abs ≤ 100,000; cannot drive Physical < Hard Locked | "Adjustment cannot drive Physical below committed Hard Locks (N units)" |
| reason | Enum | Yes | `receipt`, `count_correction`, `write_off`, `transfer_in`, `transfer_out`, `return_restock`, `damage`, `qc_resolution` | "Please select a reason" |
| note | String | No | ≤ 500 chars | "Note must be ≤ 500 characters" |
| safety_stock | Integer | Yes (for edit) | ≥ 0, ≤ Physical | "Safety Stock cannot exceed Physical" |
| low_stock_threshold | Integer | Yes (for edit) | ≥ 0 | "Threshold cannot be negative" |
| inventory_state_override | Enum | Yes (for edit) | `in_stock`, `low_stock`, `out_of_stock`, `held`, `made_to_order`, `discontinued` (per §17.4.2) | "Invalid inventory state" |
| channel_buffer (Phase 3) | Object | No | `{ channelCode, bufferQty }`; bufferQty ≥ 0 | — |
