## M04 — Checkout & Payment

---

### F-04.2 Payment Processing (Stripe)

**Summary**

Handles payment via Stripe Elements (PCI-compliant iframe). Server creates PaymentIntent with TC-16: idempotency key. TC-15: 30-second API timeout. Supports credit/debit cards, Apple Pay, Google Pay, iDEAL (NL), and SEPA Direct Debit (EU).

**Background / Context**

Payment is where purchase intent becomes revenue. Stripe handles PCI DSS complexity — no card data touches HomeStyle servers. For a premium design platform accepting high-value orders across US and EU, reliable payment processing with 3DS support, multi-currency, and EU payment methods is essential.

**User Story**

As a customer at payment, I want to securely enter my payment details and complete my purchase so my order is confirmed without my card data touching HomeStyle servers.

**Flow — Happy Path**

1. Customer arrives at payment step. Order summary shown with items, configuration details, lead times, shipping, and total.
2. 2. Frontend loads Stripe Elements (PCI-compliant — no payment data touches HomeStyle servers).
   3. 3. Server creates TC-16: Stripe PaymentIntent with idempotency key (UUID tied to checkout sessionId, 24-hour window). Trade accounts: VAT-exempt flag passed where applicable.
      4. 4. Customer enters payment details, clicks 'Place Order'.
         5. 5. Frontend calls stripe.confirmPayment(). Stripe handles 3DS if required.
            6. 6. TC-15: 30-second timeout on Stripe confirmation response.
               7. 7. On success: payment_intent.succeeded webhook received. Order record created. Inventory holds converted to permanent decrements. Cart cleared. Redirect to confirmation.
                 
                  8. **Flow — Edge Cases**
                 
                  9. | Edge Case | Expected System Behaviour |
                  10. |---|---|
                  11. | Payment declined | Show Stripe decline reason. Allow retry with same/different payment method. |
                  12. | TC-15: Stripe API timeout (30 seconds) | Show: 'Payment confirmation taking longer than expected. Check Order History before retrying to avoid duplicate charges.' Do NOT retry automatically. |
                  13. | 3DS authentication abandoned | Payment fails. Hold remains. Customer can retry. |
                  14. | Double-submit (TC-16) | Idempotency key returns original PaymentIntent result. No duplicate charge. |
                  15. | Trade VAT-exempt checkout (OD-06) | PaymentIntent created with metadata: { vatExempt: true, vatNumber: '[verified VAT#]' } for invoice generation. |
                 
                  16. **Business Rules**
                 
                  17. | BR # | Rule |
                  18. |---|---|
                  19. | BR-01 | TC-15: Stripe API timeout 30 seconds. Show error and do NOT auto-retry. |
                  20. | BR-02 | TC-16: Stripe idempotency key: UUID tied to checkout sessionId, valid 24 hours. |
                  21. | BR-03 | Order record created ONLY on payment_intent.succeeded webhook — not on frontend confirmation alone. |
                  22. | BR-04 | Discount code re-validated at PaymentIntent creation server-side. |
                  23. | BR-05 | No payment data stored by HomeStyle. PCI DSS compliance via Stripe. |
                  24. | BR-06 | Available payment methods per market: US (cards, Apple Pay, Google Pay); EU (cards, Apple Pay, iDEAL, SEPA Direct Debit); UK (cards, Apple Pay). |
