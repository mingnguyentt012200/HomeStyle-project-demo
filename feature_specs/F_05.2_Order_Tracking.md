## M05 — Order Management

---

### F-05.2 Order Tracking

**Summary**

Customers view real-time carrier tracking status for dispatched orders from their order detail page. For white-glove deliveries, appointment scheduling information is also shown.

**User Story**

As a customer waiting for my premium furniture delivery, I want to see the latest tracking status and any white-glove delivery appointment directly from my order page so I don't need to contact support.

**Flow — Happy Path**

1. Admin marks order as dispatched and enters carrier + tracking number (F-05.3).
2. 2. System updates order to status='dispatched'. TC-18: Dispatch email sent.
   3. 3. Customer views order detail → 'Track your order' link shown with carrier name.
      4. 4. GET /orders/:orderId/tracking returns latest tracking events.
         5. 5. For white-glove deliveries: delivery appointment window shown if available from white-glove partner API.
           
            6. **Flow — Edge Cases**
           
            7. | Edge Case | Expected System Behaviour |
            8. |---|---|
            9. | Carrier API unavailable | Last cached tracking data shown with timestamp. 'Tracking information may be delayed.' |
            10. | Tracking not yet active in carrier system | 'Tracking information is not yet available. Please check back in a few hours.' |
            11. | Order not yet dispatched | Tracking link not shown on order detail page. |
           
            12. **Business Rules**
           
            13. | BR # | Rule |
            14. |---|---|
            15. | BR-01 | Tracking data cached in order_tracking_events table, refreshed every 2 hours via BullMQ job. |
            16. | BR-02 | Supported carriers configured in admin settings. |
            17. | BR-03 | TC-18: Dispatch email sent within 48 hours of order confirmation. |
