### F-03.1 Shopping Cart

**Summary**

Manages product selection for both guests (Redis session) and registered customers (DB), with configuration-aware cart lines, real-time stock validation, and quantity controls.

**Background / Context**

For premium design furniture, a cart item is not just a product but a specific configuration — a sofa in walnut base with Grade B linen upholstery in Large. The cart must store and display the full configuration detail so customers can review exactly what they are buying before checkout.

**User Story**

As a customer, I want to add configured products to my cart and have them saved — including all my chosen options — so I can continue browsing and checkout when ready.

**Flow — Happy Path**

1. Customer clicks 'Add to Cart' with all configuration options selected.
2. 2. Guest: item added to Redis session (TC-12: 30-day TTL). Registered: upserted to cart_items table.
   3. 3. Cart icon updates. Slide-in drawer confirms item added: product name, configuration summary (e.g. 'Walnut / Natural Oil / Grade B Linen / Large'), price, lead time.
      4. 4. Customer views /cart: itemised list with product images, full configuration detail, quantity controls, unit price, lead time per line, line totals, subtotal.
         5. 5. Customer proceeds to checkout — TC-13: inventory hold created on checkout initiation.
           
            6. **Flow — Edge Cases**
           
            7. | Edge Case | Expected System Behaviour |
            8. |---|---|
            9. | Same configuration added twice | Quantity incremented on existing line — no duplicate line. |
            10. | Configuration goes out of stock after adding | On /cart page load: re-check stock. If 0: 'This configuration is no longer available' in red. Checkout disabled for that line. |
            11. | Quantity set to 0 via minus button | Item removed from cart automatically. |
            12. | Guest cart cookie deleted | Guest cart lost. No recovery. Customer must re-add items. |
           
            13. **Business Rules**
           
            14. | BR # | Rule |
            15. |---|---|
            16. | BR-01 | TC-12: Guest cart stored in Redis with 30-day TTL from last activity. Each update refreshes TTL. |
            17. | BR-02 | Maximum 10 units per configuration per cart. |
            18. | BR-03 | Inventory NOT decremented on add-to-cart. Only soft-reserved when checkout begins (TC-13, F-04.1). |
            19. | BR-04 | Cart totals shown are estimates (no tax). Final total calculated at checkout (F-04.4). |
            20. | BR-05 | Lead time per configuration displayed on each cart line — prominently, as this is a key purchase consideration for design furniture buyers. |
           
            21. **Input Validation Rules**
           
            22. | Field | Type | Required | Validation Rules | Error Message Shown |
            23. |---|---|---|---|---|
            24. | productId | UUID | Yes | Must be valid, non-archived product. | 422: 'Product not found or unavailable' |
            25. | configurationId | UUID | Yes | Must belong to the specified productId. | 422: 'Please select all product options' |
            26. | quantity | Integer | Yes | 1–10. Must not exceed available stock minus active holds. | 'Quantity must be between 1 and 10' / 'Only [N] available' |
