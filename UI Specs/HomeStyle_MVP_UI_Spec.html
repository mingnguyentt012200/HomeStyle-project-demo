5. Animation System

Animations serve purpose — they communicate state, guide attention, confirm action. No decoration-only animations. All transitions must respect prefers-reduced-motion: replace movement with instant opacity changes when set.

Transition Tokens

TokenValueUse Case
--t-fast100ms easeHover color/bg changes, icon swaps
--t-base180ms easeFocus ring, button state, field border
--t-slow300ms easePanel slides, dropdown open, modal enter
--t-spring400ms cubic-bezier(0.34,1.56,0.64,1)Cart add confirm bounce, success checkmark, toast enter

Named Animations

Page Load — Staggered Reveal CSS only
Trigger: page mount
Effect: elements fade in + translateY(16px → 0)
Duration: 400ms ease-out
Stagger: children receive animation-delay: 0ms, 80ms, 160ms, 240ms, 320ms
Applies to: hero text, nav, first section, product grid rows
Add to Cart — Confirmation Pulse
Trigger: POST /cart success response
Effect: button background flashes --hs-success for 600ms, then returns to --hs-clay. Cart icon in nav bounces: scale(1 → 1.3 → 1) spring. Cart count badge updates with scale(0 → 1) spring.
Duration: button 600ms, icon 400ms spring
Toast: "Item added to cart" slides up from +20px, fade in 200ms
Image Gallery — Crossfade
Trigger: thumbnail click or swipe
Effect: outgoing image fades to opacity 0 over 200ms. Incoming image fades from opacity 0 over 200ms. No translateX — avoid nausea on furniture (complex patterns).
Thumbnail: active thumbnail gets 2px --hs-clay border, scale(1.05) with 180ms transition
360° Image Carousel — Drag Scrub
Trigger: pointer drag or touch swipe
Effect: frame advances/reverses on deltaX. No easing — 1:1 with drag. On release: snap nearest frame, 100ms ease snap. Cursor: grab → grabbing. On mobile: touch-action: pan-y to prevent page scroll conflict.
Modal — Enter / Exit
Enter: backdrop fades in opacity(0 → 0.5) 200ms. Panel slides up translateY(24px → 0) + opacity(0 → 1) 300ms ease-out
Exit: reverse — 200ms ease-in. Backdrop exits last.
Backdrop: rgba(28,37,48,0.5) blur(2px)
Toast — Enter / Exit
Enter: translateY(16px → 0) + opacity(0 → 1) 200ms ease-out
Exit: translateX(0 → 100%) 180ms ease-in (slides right out), OR opacity(1 → 0) if reduced-motion
Stack: existing toasts shift up by (toast-height + 8px) with 200ms ease transition on transform
Accordion / FAQ — Expand
Effect: height: 0 → auto via max-height trick (max-height: 0 → 1000px, overflow: hidden) 280ms ease. Chevron rotates 0 → 180deg 200ms ease.
Note: Use CSS grid rows for height animation on modern browsers (grid-template-rows: 0fr → 1fr) for smoother result.
Checkout Step Progress — Stepper
Trigger: "Continue" CTA click — step validated and complete
Effect: completed step dot color transitions from --hs-clay to --hs-success over 300ms. Connecting line fills left-to-right with a width transition 400ms ease. New active step dot pulses once: scale(1 → 1.15 → 1) 300ms spring.
Content panel: old step fades out + translateX(-20px) 200ms ease-in. New step fades in + translateX(20px → 0) 250ms ease-out. 80ms gap between exit and entrance.
Wishlist Heart — Toggle
Add: heart icon scale(1 → 1.35 → 1) spring 350ms. Fill color transitions transparent → --hs-clay 200ms. Particle burst: 6 small dots radiate outward fade out 400ms.
Remove: fill color transitions --hs-clay → transparent 150ms. No scale.
Filter Panel — Slide In (Mobile)
Trigger: "Filter" button tap on mobile
Effect: full-screen panel translateX(100% → 0) 280ms ease-out. Backdrop fades in. Dismiss: swipe right or tap backdrop.
Reduced Motion — Required Implementation Every component using transform or opacity transitions must check @media (prefers-reduced-motion: reduce). Inside this query: set all transition durations to 0ms and remove keyframe animations. Replace with instant opacity changes only. This is mandatory for WCAG 2.3.3.

6. Validation Rules

Strategy: Validate on blur (not on keystroke) for text fields. Validate on change for dropdowns and checkboxes. Re-validate on submit. Show inline errors below the field with a ⚠ icon prefix. Clear error on valid input (on keystroke, after initial error shown). Never show generic "field required" — always provide specific guidance.

Error copy tone Be human, not robotic. "Please enter a valid email" ✓. "Invalid email format" ✗. "Password must have at least 8 characters" ✓. "Password too short" ✗.

Auth Fields

First NameRequired · 1–50 chars · letters, hyphens, apostrophes, spaces only

7. User Roles (Customer-Side)

RoleAuth stateSessionCapabilities
Guest / Visitor Unauthenticated Server-side session token (cookie), preserved until browser close or 30-day expiry Browse, search, view PDPs, check delivery availability, view reviews, add to cart (session-based), view wishlist if shared link, contact form
Registered Customer JWT (1hr) + silent refresh JWT in HTTP-only cookie. Refresh token 30 days. All guest permissions + persistent cart (merged from guest on login), wishlist, order history, order tracking, returns portal, post-purchase reviews, account management, saved addresses
Cart merge behavior on login When a guest user logs in or creates an account, their session cart is merged with any existing persistent cart. Conflicts (same SKU): use higher quantity. Guest cart then destroyed.
