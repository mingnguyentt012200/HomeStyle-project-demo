1. Admin Panel Overview

The HomeStyle Admin Panel is a React SPA serving as the back-office for all operational, content, and configuration tasks. It is entirely separate from the customer-facing storefront but shares the same design tokens and component library. It communicates exclusively with the backend via REST API (/api/v1/admin/*), with all endpoints requiring a valid admin JWT and role authorization.

Security: 2FA Mandatory All admin accounts must complete TOTP-based two-factor authentication on every login session. 2FA cannot be bypassed or disabled for any role. See SC-A-02 for the full auth flow.

2. Admin Roles & Permissions

Three roles exist in Phase 1, all enforced server-side. The UI hides inaccessible modules from the side navigation. Attempting to access a restricted URL redirects to a 403 page.

RoleDescriptionModule AccessFinancial AccessSystem Settings
Super AdminFull access to all admin panel modulesAll modules✓ Full✓ Full
Admin — OperationsDay-to-day order, inventory, returns, shipping, and customer managementDashboard (limited), Orders, Inventory, Returns, Customers, Shipping, Reports✗ No✗ No
Admin — ContentProduct catalogue, CMS, promotions, reviews, email templatesDashboard (limited), Products, Promotions, Reviews, CMS, Email Templates✗ No✗ No
Role enforcement is server-side only. The UI hides inaccessible navigation items for usability, but the API enforces role permissions on every request. A hidden menu item does not grant security — role validation occurs at the API layer on every request.

SC-A-30: Configures which product categories trigger LTL carrier rates (e.g. Sofas, Beds, Wardrobes). LTL rates fetched from carrier API (EasyPost / ShipEngine) or manual fallback table if API unavailable. TC-15: 30-second API timeout.
